SR-тригер (тригер Встановлення-Скидання)

Що таке SR-тригер?

SR-тригер (тригер Встановлення-Скидання) — це основна схема цифрової логіки, яка може зберігати один біт інформації. Це базовий будівельний блок елементів пам'яті в цифрових системах. SR-тригер має два входи (Встановлення та Скидання) та два виходи (Q та Q-not), що дозволяє йому зберігати стабільний стан до отримання команди на зміну.

Принцип роботи

SR-тригер зазвичай складається з двох логічних елементів АБО-НІ або двох елементів І-НІ, з'єднаних у перехресну конфігурацію:

  • Вхід Встановлення (S): При активації S (1), вихід Q стає 1 (стан ВСТАНОВЛЕННЯ)
  • Вхід Скидання (R): При активації R (1), вихід Q стає 0 (стан СКИДАННЯ)
  • Обидва входи низькі (0, 0): Тригер зберігає свій попередній стан (функція пам'яті)
  • Обидва входи високі (1, 1): Це невизначений/недійсний стан, якого слід уникати

Таблиця істинності (SR-тригер на основі АБО-НІ)

S (Встановлення) R (Скидання) Q (Вихід) Q' (Інверсний вихід) Стан
0 0 Без змін Без змін Утримання (Пам'ять)
0 1 0 1 Скидання
1 0 1 0 Встановлення
1 1 X (невизначено) X (невизначено) Недійсно

Ключові характеристики

  • Асинхронний: Вихід змінюється миттєво при зміні входів (не потребує годинника)
  • Елемент пам'яті: Зберігає стан, коли обидва входи дорівнюють 0
  • Бістабільний: Має два стабільних стани (Q=0 або Q=1)
  • Фундаментальний будівельний блок: Є основою для складніших тригерів, таких як D-тригери та JK-тригери

Застосування

  • Базові схеми пам'яті та зберігання
  • Усунення дребезжання перемикачів та кнопок
  • Логіка управління в цифрових системах
  • Основа для проектування складніших послідовних логічних схем
  • Навчальний засіб для розуміння послідовних схем
